pattern A

01 — what this file teaches

Voted, revealed.

Pattern A is the ordinary poll: someone votes, the result shows up immediately, they can change their mind, and a reload neither loses nor duplicates the vote. Use it when revealing the tally early spoils nothing.

If revealing early does spoil it — a blind test, a bake-off where seeing the leader contaminates the next vote — the pattern is B, in poll-bakeoff.html. The full bridge reference is in references/polls.md.

03 — the three things that go wrong

MistakeSymptomFix
Reading window.lattice at load The previous vote is never restored Wait for the lattice:ready event
Locking the option after the vote Nobody can change their mind Keep it clickable; the server does last-write-wins
Faking a result with no bridge Invented numbers in the loose file Degrade with an honest notice
